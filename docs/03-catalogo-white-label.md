# Catálogo White-Label

## Estratégia

Criar produtos reaproveitáveis para reduzir custo de entrega e acelerar fechamento comercial. Cada produto deve ter:

1. Escopo padrão fechado.
2. Itens opcionais de upsell.
3. Tempo de implantação previsível.
4. Personalização visual por tenant.

## Produtos iniciais sugeridos

| Produto | Nicho | Funcionalidades base |
| --- | --- | --- |
| AgendaPro | Clínicas, salões, barbearias | Agenda, lembrete, clientes, relatórios |
| MenuGo | Restaurantes e lanchonetes | Cardápio digital, pedidos, status |
| OS Manager | Oficinas e assistência | Ordem de serviço, check-in, peças, histórico |
| LeadSite | Prestadores de serviço | Landing page + mini CRM |
| SchoolLite | Cursos e escolas pequenas | Turmas, alunos, presença, cobrança simples |
| SiteCMS | Negócios locais | Site institucional com painel de conteúdo |

## Mecanismo de personalização

1. Tema por cliente: cores, logo, nome e favicon.
2. Domínio customizado por tenant.
3. Planos por recursos (ativação de módulos).
4. Permissões por perfil de usuário.

## SLA de implantação

1. Produto sem integração: até 7 dias úteis.
2. Produto com integração simples: até 15 dias úteis.
3. Produto com integração complexa: escopo adicional.

## Estratégia de evolução

1. Melhorias comuns entram no core do produto.
2. Customizações específicas ficam isoladas por tenant.
3. Features experimentais entram com feature flag.
